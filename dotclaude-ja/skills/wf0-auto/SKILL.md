# /wf0-auto

wf1〜wf5 のワークフロー全体を一発で実行する。中間の承認ポイントで停止しない。

このスキルは**オーケストレーター**である。既存の wf1〜wf5 スキルに処理を委譲し、再実装はしない。各フェーズでは対応する `/wfN-*` スキルを読み込んでその指示に従い、以下に記載する自動モードオーバーライドのみ適用する。

## このスキルの動作

以下のスキルを順番に呼び出す:

1. `/wf1-kickoff <work-id> <goal>` - ワークスペース作成、キックオフドキュメント生成
2. `/wf2-spec <work-id>` - コードベース調査、仕様書ドラフト作成
3. `/wf3-plan <work-id>` - 実装計画の作成
4. `/wf4-impl <work-id>` - 全実装ステップの実行
5. `/wf5-review <work-id>` - レビューと修正適用

各 `/wfN-*` スキルを個別実行する場合との違い: 各フェーズに**自動モードオーバーライド**を適用し、ユーザー承認ゲートがパイプラインをブロックしないようにする。

## 引数

- `<work-id>`: 作業のkebab-case識別子（必須）
- `<goal>`: 達成したいこと - タスクの明確な説明（必須、work-id以降の全て）

## ワークフロー

### フェーズ 0: 入力の検証

work-id または goal が不足している場合、両方をユーザーに尋ねる。揃わなければ開始しない。

`docs/wf/<work-id>/` が既に存在するか確認する。存在する場合は `state.json` を確認して再開判定（下記「再開動作」参照）。`state.json` がない場合はユーザーに警告し、上書きするか新しいwork-idにするか尋ねる。

### フェーズ 1: キックオフ

`/wf1-kickoff` スキル（`skills/wf1-kickoff/SKILL.md`）を読み込み、その指示に従う。

**自動モードオーバーライド:**
- ユーザーインタビュー（ステップ 4）をスキップする。代わりに、提供された `<goal>` からキックオフドキュメントの内容を導出する:
  - **ゴール**: ユーザーのgoalをそのまま使用
  - **完了基準**: goalから推測
  - **制約**: コードベースのコンテキストから推測、または最小限
  - **スコープ外**: goalに基づいて合理的な境界を設定
  - **依存関係**: コードベースで関連する依存関係を確認
  - **未解決の質問**: 空にする（自動モードではブロッカーなしと仮定）
- 完成したキックオフドキュメントのユーザー確認を求めない — 即座に続行する。

**進捗出力**: `[1/5] キックオフ完了。`

### フェーズ 2: 仕様

`/wf2-spec` スキル（`skills/wf2-spec/SKILL.md`）を読み込み、その指示に従う。

**自動モードオーバーライド:**
- ステップ 4（ユーザーレビュー）をスキップする。仕様書を承認のために提示したり反復したりしない — 最初のドラフトを受け入れて続行する。

**進捗出力**: `[2/5] 仕様完了。`

### フェーズ 3: 計画

`/wf3-plan` スキル（`skills/wf3-plan/SKILL.md`）を読み込み、その指示に従う。

**自動モードオーバーライド:**
- ステップ 4（ユーザーレビュー）をスキップする。計画をフィードバックのために提示しない — 最初のドラフトを受け入れて続行する。

**進捗出力**: `[3/5] 計画完了。Nステップ定義。`

### フェーズ 4: 実装

`/wf4-impl` スキル（`skills/wf4-impl/SKILL.md`）を読み込み、その指示に従う。

**自動モードオーバーライド:**
- 全ステップをポーズなしで実行する。各ステップ後にユーザーに続行するか聞かない — 全て連続実行する。
- ステップが失敗し、計画通りに完了できない場合:
  - 問題を `05_IMPLEMENT_LOG.md` に記録
  - 合理的な修正を試みる
  - 修正成功なら次のステップへ続行
  - 修正失敗なら記録して次へ（未完了ステップとして記録）
  - ユーザーにどうするか聞かない — ベストエフォートで判断して続行する。

**進捗出力**: `[4/5] 実装完了。N/Mステップ成功。`

### フェーズ 5: レビュー

`/wf5-review` スキル（`skills/wf5-review/SKILL.md`）を読み込み、その指示に従う。

**自動モードオーバーライド:**
- verdictが `request-changes` の場合: ユーザーの承認を求めずに非破壊的な修正を自動適用する。
- verdictが `needs-discussion` の場合: 未解決の質問を記録するがブロックしない — 最終サマリーで報告する。

**進捗出力**: `[5/5] レビュー完了。判定: <verdict>。`

### 最終レポート

全フェーズ完了後、サマリーを提示:

```
## ワークフロー完了: <work-id>

- キックオフ: 完了
- 仕様: 完了（N件の要件）
- 計画: 完了（Nステップ）
- 実装: 完了（N/Mステップ成功）
- レビュー: <verdict>

ドキュメント: docs/wf/<work-id>/
```

手動対応が必要な問題（失敗した実装ステップ、needs-discussionのレビュー指摘事項）を一覧表示する。

## エラー処理

- いずれかのフェーズで回復不能なエラーが発生した場合、関連ドキュメントに記録し、`state.json` を失敗地点を反映するよう更新し、ユーザーに報告する。エラーを黙って飲み込まない。
- ワークフローは再開可能: 途中で停止した場合、ユーザーは問題を修正して `/wf0-auto <work-id>` を再実行できる。スキルは `state.json` から現在の状態を検出し、中断箇所から再開する。

## 再開動作

`docs/wf/<work-id>/` が既に存在し `state.json` がある場合:

1. `state.json` を読んで現在のフェーズを判定
2. 完了済みフェーズをスキップ
3. 現在のフェーズ（または次の保留フェーズ）から再開し、上記の自動モードオーバーライドを適用
4. implフェーズの場合: `current_step + 1` から再開

## 重要な注意事項

- このスキルは wf1〜wf5 スキルに**処理を委譲**する。各スキルの SKILL.md を読み込んでその指示に従い、上記の自動モードオーバーライドのみ適用する。
- wf1〜wf5 スキルが更新された場合、実行時に読み込むためこのスキルも自動的にその恩恵を受ける。
- 全ドキュメントは日本語で作成する（テンプレートの言語に従う）。
- `state.json` の全タイムスタンプはUTC: `YYYY-MM-DDTHH:MM:SSZ`。
