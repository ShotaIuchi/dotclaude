# Kotlin Multiplatform コンテキストルール

KMP開発タスク時に自動適用されるコンテキスト。

## 適用タイミング

以下の作業時に適用:
- KMP共有モジュール作成
- expect/actual宣言使用
- Koin、SQLDelight、Ktor設定
- Compose Multiplatform UI実装

## 参照ドキュメント

| ドキュメント | 目的 |
|-------------|------|
| Clean Architecture | レイヤー分離の原則 |
| Testing Strategy | テストパターンとカバレッジ |
| Kotlin Conventions | 命名、構造、イディオム |
| Library Patterns | Koin、SQLDelight、Ktor使用法 |
| Feature Patterns | 機能モジュール構造 |
| KMP Architecture | マルチプラットフォームパターン |
