# /feature-team

Agent Teamsで新機能の並列実装チームを自動構成・起動する。

## 目的

- 複数の専門家による新機能の並列実装
- 機能種別に応じたスペシャリストの自動選定

## 使用方法

```
/feature-team [--issue N | --pr N | path | description]
```

## 使用例

```bash
# Issueの機能を実装
/feature-team --issue 789

# 機能の説明から実装
/feature-team "ユーザー認証機能を追加"

# 特定ディレクトリに機能を追加
/feature-team src/auth/
```

## 動作の流れ

1. リード（メインコンテキスト）が機能を分析し、スコープとコンポーネントを判定
2. 選定マトリクスに基づきスペシャリストを選定（最大7種）
3. **Task tool** で各スペシャリストをサブエージェントとして並列起動
4. 各サブエージェントの結果をリードが統合し、完了レポートを作成

## スペシャリスト一覧

| スペシャリスト | 観点 |
|-------------|------|
| API Designer | エンドポイント設計、スキーマ、バージョニング |
| UI Implementer | UIコンポーネント、レイアウト、アクセシビリティ |
| Data Modeler | データスキーマ、マイグレーション、インデックス |
| Business Logic | ビジネスルール、バリデーション、ワークフロー |
| Test Writer | ユニットテスト、インテグレーションテスト |
| Doc Writer | APIドキュメント、ユーザーガイド |
| Security Analyst | 認証、認可、入力検証、データ保護 |

## 注意事項

- 各スペシャリストは別コンテキスト（サブエージェント）で動作する
- 全サブエージェントが並列実行されるため、トークン消費が増加する
