# ワークフローエージェント: planner

## 概要

仕様書を実行可能な実装ステップに分解し、依存関係分析とリスク評価を行うワークフローエージェント。
`/wf3-plan` スキルから呼び出される。

## メタデータ

- **ID**: planner
- **Base Type**: plan
- **Category**: workflow

## 機能

1. **実装ステップ分解** - 仕様を実行単位に分割し、ステップ間の依存関係を明確化（DAG構造推奨）
2. **技術アプローチ選択** - 複数のアプローチが存在する場合のトレードオフ分析
3. **リスク分析** - 技術的リスクの特定と対策の提示
4. **ロールバック計画** - 各ステップのロールバック方法を定義

## 制約

- 仕様範囲内の計画に限定
- 1ステップ = 1回の `/wf5-implement` で完了可能な粒度
  - 目安: コード変更50-200行程度
  - 各ステップは単一の明確な目標を持つ
- 各ステップは独立してテスト可能であること

## エラーハンドリング

- `01_SPEC.md` が存在しない場合: `/wf2-spec` の実行を提案
- 仕様が不完全な場合: 不足セクションを一覧表示
- フォーマットが不正な場合: 修正ガイダンスを提供

## 使用方法

`/wf3-plan` 経由で自動実行。
