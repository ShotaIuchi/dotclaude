# Reference Decisions ルール

## 概要

`references/` の各カテゴリディレクトリには `decisions.md` を配置し、技術選定の判断記録を管理する。公式ドキュメントのリンク集（`index.md`）は使用しない。

## ファイル名規則

- `decisions.md` — 使用する（技術選定の記録であることが明確）
- `index.md` — 禁止（中身のない目次を連想、Claudeが既知の公式情報の重複になる）

## decisions.md の構成

3つの必須セクションで構成する：

1. **採用技術** — 技術名・用途・採用理由・代替候補を表形式で記載
2. **不採用とした選択肢** — 使わない技術と不採用理由を表形式で記載
3. **関連ドキュメント** — 同ディレクトリ内の conventions.md や patterns ファイルへのリンク

## ファイルの住み分け

| ファイル | 役割 | 例 |
|----------|------|-----|
| `decisions.md` | 何を使い、何を使わないか | Hilt採用、Koin不採用 |
| `conventions.md` | どう書くか | 命名規則、ディレクトリ構造 |
| `*-patterns.md` | どう組むか | MVVM/UDF実装パターン |

## 禁止事項

- `references/` 配下に `index.md` を作成しない
- 公式ドキュメントURLのリンク集を含めない
- 採用理由なしに技術名だけを列挙しない
