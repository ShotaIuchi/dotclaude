# /design-team

Agent Teamsで設計検討・複数視点議論チームを自動構成・起動する。

## 目的

- 複数の視点からの構造化された設計議論と意思決定
- 設計トピックに応じたパネリストの自動選定

## 使用方法

```
/design-team [--issue N | path | question]
```

## 使用例

```bash
# 設計質問を議論
/design-team "認証にJWTとセッションのどちらを使うべきか"

# Issueの設計を検討
/design-team --issue 321

# 既存コードのリアーキテクチャを議論
/design-team src/api/
```

## 動作の流れ

1. リード（メインコンテキスト）が設計トピックを分析し、ドメインを判定
2. 選定マトリクスに基づきパネリストを選定（最大7種）
3. **Task tool** で各パネリストをサブエージェントとして並列起動
4. 各サブエージェントの結果をリードが統合し、設計決定文書を作成

## パネリスト一覧

| パネリスト | 観点 |
|----------|------|
| Pragmatist | 実用性、チーム能力、メンテナンスコスト |
| Futurist | 長期スケーラビリティ、将来要件 |
| Skeptic | リスク、エッジケース、障害シナリオ |
| Domain Expert | 業界ベストプラクティス、参照アーキテクチャ |
| User Advocate | エンドユーザー体験、DX |
| Cost Analyst | インフラコスト、運用コスト、TCO |
| Standards Keeper | コーディング規約、既存パターンとの一貫性 |

## 注意事項

- 各パネリストは別コンテキスト（サブエージェント）で動作する
- 全サブエージェントが並列実行されるため、トークン消費が増加する
