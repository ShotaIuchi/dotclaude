## 概要 (Summary)

このドキュメントは `doc-reviewer` エージェントの定義ファイルである。単一のドキュメントファイルをレビューし、`.review.md` ファイルを生成するタスクエージェントを定義している。`/doc-review` コマンドから呼び出され、複数ファイルの並列処理に対応する設計となっている。

主な機能:
- ドキュメント分析（目的・完全性・明確性・一貫性の評価）
- 技術的正確性のレビュー
- 優先度付き改善提案の生成

- [x] **明確性 (Clarity)**: 読者にとって分かりやすい

- [x] **情報の正確性 (Correct information)**: 記載内容が正確

## 総評 (Overall Assessment)

`doc-reviewer.md` は、タスクエージェントの定義として高品質なドキュメントである。

**良い点:**
- メタデータ、目的、コンテキスト、機能、制約、手順、出力形式が体系的に整理されている
- 疑似コードと具体例により、実装者が理解しやすい構成になっている
- 単一責任の原則に従い、1ファイルのみを処理するという明確なスコープが定義されている
- テンプレートのプレースホルダーと評価観点が表形式で分かりやすく記載されている

**改善余地:**
- 一部のエッジケース（テンプレート不在、既存ファイル上書き）の処理が未定義
- JSON出力とMarkdownファイル出力の関係性がやや曖昧

全体として、このドキュメントはプロジェクトの品質基準を満たしており、エージェントの実装・運用に十分な情報を提供している。中程度の改善点を反映することで、さらに堅牢なドキュメントとなる。

**推奨アクション**: 中優先度の改善点を次回のドキュメント更新時に反映することを推奨する。
